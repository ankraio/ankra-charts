3. Create a ClusterTunnel pointing at your Cloudflare account. The Secret
   `{{ include "cloudflare-operator.cloudflareSecretName" . }}` is expected to live in
   `{{ include "cloudflare-operator.namespace" . }}` and contain keys
   `{{ .Values.credentials.apiTokenKey }}` and `{{ .Values.credentials.accountIdKey }}`.
{{- if .Values.credentials.create }}

   This release rendered the Secret with the values you supplied.
{{- else }}

   You set `credentials.existingSecret={{ .Values.credentials.existingSecret }}` — make
   sure that Secret exists in `{{ include "cloudflare-operator.namespace" . }}` before
   the operator can reconcile tunnels.
{{- end }}

   Minimal example (TunnelBinding glues a Service to a hostname):

       apiVersion: networking.cfargotunnel.com/v1alpha2
       kind: ClusterTunnel
       metadata:
         name: prod-tunnel
       spec:
         newTunnel:
           name: prod-tunnel
         cloudflare:
           email: admin@example.com
           domain: example.com
           secret: {{ include "cloudflare-operator.cloudflareSecretName" . }}
           accountId: <your-cloudflare-account-id>
       ---
       apiVersion: networking.cfargotunnel.com/v1alpha1
       kind: TunnelBinding
       metadata:
         name: my-app
         namespace: my-app
       subjects:
         - name: my-app          # Service name
           spec:
             fqdn: app.example.com
       tunnelRef:
         kind: ClusterTunnel
         name: prod-tunnel
